1
Buat repositori di GitHub
~10 menit · Ini "lemari" tempat kode kamu disimpan — Vercel akan ambil kode dari sini
🐙
Langkah 1.1

Buat akun GitHub (kalau belum punya)

  1. Buka github.com → klik Sign up
  2. Daftar pakai email, pilih username, verifikasi email
  3. Pilih paket Free — cukup untuk proyek ini
📁
Langkah 1.2

Buat repositori baru

  1. Login GitHub → klik tombol + (pojok kanan atas) → New repository
  2. Isi Repository name: pak-andung
  3. Pilih Private (agar kode tidak terlihat publik)
  4. Centang Add a README file
  5. Klik Create repository
⬆️
Langkah 1.3

Upload semua file ke GitHub

Cara termudah tanpa perlu paham Git:

  1. Di halaman repositori → klik Add file → Upload files
  2. Drag & drop seluruh folder pak-andung ke area upload
  3. Scroll ke bawah → tulis pesan commit: initial: pak andung app
  4. Klik Commit changes
⚠️ Jangan pernah upload file iniFile .env.local (berisi password & API key) TIDAK boleh di-upload ke GitHub. File .gitignore yang sudah ada seharusnya memproteksinya secara otomatis. Double-check: pastikan kamu tidak melihat file .env.local di repositori GitHub setelah upload.
2
Setup Supabase (database + login)
~15 menit · Supabase menyimpan data pembeli, login, dan riwayat chat
🗄️
Langkah 2.1

Buat akun & project Supabase

  1. Buka supabase.com → klik Start your project → login pakai GitHub
  2. Klik New Project
  3. Isi Name: mulaiinvest-pakandung
  4. Buat Database Password yang kuat — simpan di tempat aman (tidak perlu dipakai di env variable)
  5. Pilih Region: Southeast Asia (Singapore) — paling dekat dengan Indonesia
  6. Klik Create new project → tunggu ~2 menit sampai siap
🏗️
Langkah 2.2

Jalankan skema database

Ini akan membuat semua tabel yang dibutuhkan (profil pengguna + riwayat chat).

  1. Di dashboard Supabase → klik menu SQL Editor (ikon database di sidebar kiri)
  2. Klik New query
  3. Copy seluruh isi file supabase-schema.sql → paste ke editor
  4. Klik tombol Run (atau Ctrl+Enter)
  5. Pastikan hasilnya hijau semua — tidak ada error merah
Cara cek berhasilSetelah run, klik menu Table Editor di sidebar. Kamu harus melihat 2 tabel: profiles dan chat_messages. Kalau sudah ada, berarti sukses.
🔑
Langkah 2.3

Ambil API Keys Supabase

Kamu butuh 3 angka dari Supabase:

  1. Di Supabase → klik ikon roda gigi (⚙️) Settings → API
  2. Catat Project URL → ini akan jadi NEXT_PUBLIC_SUPABASE_URL
  3. Di bagian "Project API keys" → catat anon public → ini NEXT_PUBLIC_SUPABASE_ANON_KEY
  4. Catat service_role secret (klik "Reveal") → ini SUPABASE_SERVICE_ROLE_KEY — jaga kerahasiaan ini!
Service role key sangat sensitifKey ini bisa bypass semua keamanan database. Jangan pernah taruh di kode yang bisa dilihat publik atau di frontend. Hanya dipakai di webhook yang berjalan di server.
📧
Langkah 2.4

Nonaktifkan konfirmasi email Supabase

Secara default, Supabase minta pembeli verifikasi email sebelum bisa login. Kita akan skip ini karena akun dibuat oleh webhook (bukan pembeli sendiri).

  1. Supabase → Authentication → Providers
  2. Klik Email
  3. Matikan toggle Confirm email → klik Save
3
Setup Resend (email otomatis)
~20 menit · Resend mengirim email login ke pembeli setelah bayar
📨
Langkah 3.1

Daftar Resend & verifikasi domain

  1. Buka resend.com → Sign up (gratis, 3.000 email/bulan)
  2. Setelah login → klik Domains → Add Domain
  3. Masukkan mulaiinvest.id → klik Add
  4. Resend akan menampilkan beberapa DNS records (3–4 baris)
  5. Login ke panel Hostinger → Domains → mulaiinvest.id → DNS / Nameservers
  6. Tambahkan setiap record dari Resend satu per satu ke DNS Hostinger
  7. Kembali ke Resend → klik Verify → tunggu ~10–30 menit sampai status Verified
Kenapa perlu verifikasi domain?Ini membuktikan ke server email dunia bahwa kamu yang punya mulaiinvest.id dan berhak kirim email dari sana. Tanpa ini, email akan masuk spam pembeli.
🔑
Langkah 3.2

Ambil API Key Resend

  1. Di Resend → klik API Keys → Create API Key
  2. Beri nama: pak-andung-production
  3. Permission: Sending access
  4. Klik Add → langsung copy key-nya sekarang — tidak akan ditampilkan lagi
4
Deploy ke Vercel + isi semua environment variables
~20 menit · Ini yang membuat app.mulaiinvest.id hidup di internet
▲
Langkah 4.1

Import project dari GitHub ke Vercel

  1. Login ke vercel.com dengan akun yang sudah kamu punya
  2. Klik Add New… → Project
  3. Klik Import Git Repository → pilih pak-andung
  4. Vercel akan otomatis mendeteksi ini adalah proyek Next.js — biarkan semua setting default
  5. JANGAN klik Deploy dulu — lanjut ke Langkah 4.2 untuk isi env variables
🔐
Langkah 4.2

Isi Environment Variables

Di halaman yang sama (sebelum klik Deploy) → expand bagian Environment Variables → tambahkan satu per satu:

SUPABASE_SERVICE_ROLE_KEYwajib service_role key dari Supabase Settings → API

🚀
Langkah 4.3

Deploy!

  1. Setelah semua env variable terisi → klik Deploy
  2. Tunggu ~2–3 menit — Vercel akan membangun app-mu
  3. Kalau ada error merah, screenshot dan tanyakan ke saya (Claude) — saya akan bantu debug
  4. Kalau berhasil (centang hijau) → Vercel akan beri URL sementara seperti pak-andung-abc123.vercel.app
🌐
Langkah 4.4

Sambungkan subdomain app.mulaiinvest.id

  1. Di Vercel → project kamu → Settings → Domains
  2. Klik Add → ketik app.mulaiinvest.id → klik Add
  3. Vercel akan menampilkan DNS record yang harus ditambahkan
  4. Login Hostinger → DNS mulaiinvest.id → tambahkan record CNAME:
    Name: app → Value: cname.vercel-dns.com
  5. Tunggu propagasi DNS (5 menit – 1 jam)
  6. Setelah selesai, https://app.mulaiinvest.id akan menampilkan halaman login Pak Andung 🎉
5
Sambungkan Lynk.id ke webhook
~10 menit · Ini yang membuat akun pembeli terbuat otomatis setelah bayar
🔗
Langkah 5.1

Daftarkan URL webhook di Lynk.id

  1. Login ke dashboard Lynk.id → Settings atau Pengaturan
  2. Cari menu Webhook atau Integrasi
  3. Masukkan URL webhook kamu:
    https://app.mulaiinvest.id/api/lynk-webhook
  4. Pilih event: payment.success atau setara
  5. Simpan
Catatan penting soal webhook LynkSetiap platform punya cara berbeda. Kalau kamu tidak menemukan menu "Webhook" di Lynk, cek dokumentasi Lynk.id atau hubungi support mereka dan minta cara untuk mendapatkan notifikasi POST saat pembayaran sukses.
🧪
Langkah 5.2

Test webhook dengan simulasi

Sebelum launch, test dengan data dummy dulu:

Test via curl (jalankan di terminal / pakai Postman)
curl -X POST https://app.mulaiinvest.id/api/lynk-webhook \ -H "Content-Type: application/json" \ -d '{ "event": "payment.success", "buyer_name": "Budi Tes", "buyer_email": "budi.tes@email.com", "product_name": "Pak Andung — Akses 30 Hari", "amount": 79000 }'

Kalau berhasil: akun budi.tes@email.com terbuat di Supabase, dan email login terkirim. Cek di Supabase → Table Editor → profiles.

Tidak bisa pakai curl? Pakai Hoppscotch (gratis, browser)Buka hoppscotch.io → Method: POST → URL: (webhook URL kamu) → Body: JSON (isi seperti di atas) → Send.
🔒
Langkah 5.3 (opsional tapi disarankan)

Aktifkan verifikasi secret webhook

Setelah semua berjalan, buka file app/api/lynk-webhook/route.ts dan hapus // (uncomment) di bagian verifikasi secret:

Uncomment bagian ini di lynk-webhook/route.ts
// HAPUS '//' di baris ini untuk mengaktifkan keamanan: // const secret = req.headers.get('x-webhook-secret') // if (secret !== process.env.LYNK_WEBHOOK_SECRET) { // return NextResponse.json({ error: 'Unauthorized' }, { status: 401 }) // } // Menjadi seperti ini (hapus // di awal setiap baris): const secret = req.headers.get('x-webhook-secret') if (secret !== process.env.LYNK_WEBHOOK_SECRET) { return NextResponse.json({ error: 'Unauthorized' }, { status: 401 }) }

Lalu daftarkan LYNK_WEBHOOK_SECRET yang sama di setting Lynk.id (bagian "Webhook Secret" atau sejenisnya).

Cara mengedit kepribadian Pak Andung
Satu file, satu tempat — tidak perlu menyentuh file lain

Semua kepribadian, gaya bicara, dan aturan Pak Andung ada di satu file: lib/system-prompt.ts

Buka file itu di GitHub (klik nama file → klik ikon pensil ✏️ di kanan atas) dan edit bagian SYSTEM_PROMPT. Setelah simpan → Vercel otomatis redeploy dalam 1–2 menit.

Yang paling sering perlu diedit: - Nama bisnis atau produk yang disebut Pak Andung
- Saham atau sektor yang jadi fokus
- Nada bicara (lebih formal / lebih santai)
- Email admin untuk pertanyaan teknis
- Konten 4 dimensi analisa saham

✅ Checklist Sebelum Launch

  • GitHub repository pak-andung sudah dibuat (private)
  • Semua file kode sudah di-upload ke GitHub
  • File .env.local TIDAK ada di GitHub (terlindungi oleh .gitignore)
  • Supabase project dibuat di region Singapore
  • SQL schema sudah dijalankan — tabel profiles dan chat_messages sudah ada
  • Email confirmation di Supabase Auth sudah dimatikan
  • Domain mulaiinvest.id sudah terverifikasi di Resend
  • Semua 7–9 environment variables sudah diisi di Vercel
  • Deploy Vercel berhasil (tidak ada error merah)
  • Subdomain app.mulaiinvest.id sudah diarahkan ke Vercel via CNAME
  • Halaman login muncul di app.mulaiinvest.id ✓
  • Test webhook manual berhasil — akun dibuat + email terkirim ✓
  • Test login + chat Pak Andung berjalan normal ✓
  • Test akun dengan access_expires_at kemarin → tampil halaman "Akses berakhir" ✓
  • URL webhook didaftarkan di Lynk.id
  • Test beli produk Lynk (Rp 0 trial) → akun terbuat → email terkirim ✓
🆘 Kalau ada error atau macet di langkah mana pun Jangan panik. Buka claude.ai, ketik: "Saya sedang setup Pak Andung di Vercel dan dapat error ini: [paste error-nya]" — saya akan bantu debug langkah demi langkah. Error saat deploy adalah hal normal, bukan tanda kamu melakukan sesuatu yang salah.
